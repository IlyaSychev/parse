Создание скрипта grib_filter на сервере opn NOCEP nomads
https://nomads.ncep.noaa.gov
Автоматизация загрузок с помощью grib_filter
 
Введение

Опция grib_filter на https://nomads.ncep.noaa.gov используется для загрузки подмножеств прогнозов NCEP. Как следует из названия, опция работает с файлами grib (версии 2) и является фильтром. Эта опция позволяет выбирать региональные подмножества, а также уровень и переменные. Поскольку разрешение модели растет быстрее, чем пропускная способность, или деньги, которые приходится платить за пропускную способность, пользователям придется начать использовать такие средства, как grib_filter.

Grib_filter - это интерактивное средство для загрузки подмножеств данных, поэтому оно кажется неподходящим для загрузки более нескольких файлов. Однако мы были хитры. Как только вы узнаете, как загружать данные в интерактивном режиме, вы нажимаете кнопку и генерируете волшебный URL. Как только вы сгенерируете этот волшебный URL, вы передаете URL в curl или wget для загрузки данных. Используя скрипт scripting 101, вы можете написать скрипт для загрузки данных за другое время и прогнозируемые часы. Используя cronjobs 101, вы можете запускать этот скрипт каждый день и автоматически получать ежедневные прогнозы.

Во всех моих примерах будут использоваться оболочка bash и программа curl. Процедура проста, поэтому многие пользователи реализовали процедуру на разных языках программирования.

Для моего примера я использую прогнозы GFS 1x1 на https://nomads.ncep.noaa.gov. Я выбрал каталог gfs.2013072218 и файл gfs.t18.pgrbf00.grib2. Затем я выбрал 3 уровня: 500, 700 и 900 мб. 5 переменных: HGT, RH, TMP, UGRD, VGRD. Я включил опцию subregion ("создать subregion") и выбрал домен 20N-60N, 250E-330E. Наконец, я выбираю опцию "Показывать URL только для веб-программирования".


Когда я нажимаю "Загрузить", я получаю волшебный URL:

URL= https://nomads.ncep.noaa.gov/cgi-bin/filter_gfs.pl?file=gfs.t18z.pgrbf00.grib2&lev_500_mb=on&lev_700_mb=on&lev_1000_mb=on&var_HGT=on&var_RH=on&var_TMP=on&var_UGRD=on&var_VGRD=on&subregion=&leftlon=250&rightlon=330&toplat=60&bottomlat=20&dir=%2Fgfs.2013072218

Если вы изучите приведенный выше URL, то увидите, что аргументы начинаются после вопросительного знака и разделяются амперсандами (&). Вы также можете заметить, что косые черты заменены на %2F . Все это стандартные вещи при работе с URL-адресами. Если вы присмотритесь, то увидите имя обрабатываемого файла (file=gfs.t18z.pgrbf00.grib2), а также каталог (dir=%2Fgfs.2013082218). (%2F переводится как / .) Вот прототип скрипта для "выполнения" URL-адреса и загрузки файла. В скрипте я заменил коды даты и прогнозируемый час переменными.

#!/bin/sh
#
# определить URL
#
fhr=00
hr=18
дата =20130821
URL="https://nomads.ncep.noaa.gov/cgi-bin/filter_gfs.pl?\
file=gfs.t ${hr}z.pgrbf ${fhr}.grib2&\
lev_500_mb=on&lev_700_mb=on&lev_1000_mb=on&\
var_HGT=on&var_RH=on&var_TMP=on&var_UGRD=on&\
var_VGRD=on&subregion=&leftlon=250&\
rightlon=330& toplat=60&bottomlat=20&\
dir=%2Fgfs.${дата} ${hr}"

# загрузить файл
curl "$URL" -o download.grb
# добавить режим ожидания, чтобы предотвратить отказ в обслуживании в случае отсутствия файла
режим ожидания 1

Я скопировал приведенный выше скрипт в файл, изменил код даты, изменил его на 755. Он запустился. Если он не запустится, в файле download.grb появится сообщение об ошибке.

Преобразование вышеупомянутого скрипта для загрузки требуемых прогнозируемых часов, запуска с текущей датой и запуска в качестве задания cron - это создание скрипта 101. Возникли проблемы, обратитесь за помощью к местному специалисту по написанию сценариев. Хотите преобразовать приведенный выше скрипт bash в Windows? Я уверен, что это можно сделать, и гуру Windows может помочь.

Совместимость: в качестве критерия проектирования новые версии grib_filter не должны нарушать работу существующих пользовательских скриптов. API не должен меняться между версиями grib_filter. Однако со временем модель будет развиваться, как и доступные поля. Это может привести к поломке пользовательских скриптов.

Комментарии: текущий grib-фильтр (1.1.0.beta-6) на nomads.ncep.noaa.gov был написан еще в июне 2008 года. Следующая версия grib-filter позволит выполнять интерполяцию в разные сетки и извлекать значения в разных местоположениях. API будет таким же, за исключением того, что переменные и уровни нужно будет выбирать из меню. (Это всегда имело место в операционных nomads, но не в nomdads разработки.)

Снимок экрана основан на nomads 2013-08. Будущие экраны могут выглядеть иначе.

Вопросы?

Сайт, nomads.ncep.noaa.gov, находится в ведении NCEP Central Operations (NCO). Вопросы следует направлять NCO через службу поддержки noamds. Джун Ван и Уэсли Эбисузаки были ответственны за написание оригинального grib_filter (6/2008), который NCO адаптировало для своего сайта nomads. Ни Джун, ни Уэсли не участвуют в разработке nomads.ncep.noaa.gov.

Вопросы? https://nomads.ncep.noaa.gov/txt_descriptions/Help_Desk_doc.shtml

  1. Ошибка: wget -O outputfile https://nomands.ncep.noaa.gov /(остальная часть строки)
    • Добавить кавычки: wget -O outputfile "https://nomands.ncep.noaa.gov /(остаток строки)"
  2. Ошибка: выходной файл curl -o https://nomands.ncep.noaa.gov /(остальная часть строки)
    • Добавить кавычки: выходной файл curl -o "https://nomands.ncep.noaa.gov /(остальная часть строки)"
  3. grib_filter возвращает все поле без подмножества
    • запрошенное поле находится за пределами домена поля данных
    • запрошенное поле слишком маленькое; в поле нет точки сетки
    • неверное указание поля, левая долгота < правая долгота
      для перехода слева = 10 ВТ направо = 10E, используйте left= 350 и right= 370
      для перехода слева = 10E направо = 10W, используйте left=10 и right= 350
    • grib_filter настроен некорректно (проблема с конфигурацией)
    • wgrib2 не поддерживает тип сетки ввода (проблема с конфигурацией)
  4. Почему grib_filter не обрабатывает XYZ? Данные уже есть на сайте.
    • Обратитесь в службу поддержки nomads.
  5. Подмножество занимает больше места на диске, чем не относящееся к подмножеству grib
    • Сжатие Grib может быть включено в конфигурации.
    • Попросите службу поддержки nomads включить сжатие grib.
  6. Загрузка происходит медленно, время сеанса загрузки истекло
    • Обратитесь за помощью в службу поддержки nomads
    • Разбейте запрос на два меньших запроса
    • Проблема должна быть устранена, когда входной файл переходит от медленного сжатия JPEG2000 к сложной упаковке.
  7. Запрашивает, чтобы grib-filter обрабатывал несколько файлов.
    • Я не думаю, что это хорошая идея. Вам не нужна единственная команда, которая может получить доступ к TBS данных (подумайте о повторном анализе).
    • Научитесь писать сценарии для нескольких запросов.
  8. Почему grib_filter все еще находится в бета-версии? Изменится ли это когда-нибудь?
    • 4/2016: Код не обновлялся с тех пор, как он был впервые установлен еще в 2008 году. Я слышал, что NCO хочет изменить код с perl на php.
  9. Доступен ли исходный код grib_filter?
    • Я не могу предоставить вам форк 2008 года, который использует NCO.
    • Я могу предоставить вам версию кода для разработки nomads (2014).
  10. Как работает grib_filter?
    • Grib_filter - это perl-скрипт, генерирующий веб-страницы и вызывающий wgrib2 для создания подмножества.
